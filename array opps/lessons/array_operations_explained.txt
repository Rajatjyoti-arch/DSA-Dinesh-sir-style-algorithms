================================================================================
     ARRAY OPERATIONS — DEEP DIVE (How & Why Each One Works)
================================================================================
  After reading this, you will understand every array operation from the
  INSIDE OUT — not just the steps, but WHY each step exists.
================================================================================


================================================================================
OPERATION 1: TRAVERSAL
================================================================================

WHAT IT DOES:
  Visit every single element in the array, one by one, from first to last.

WHY YOU NEED IT:
  It's the foundation of EVERYTHING. Searching, counting, summing, printing
  — they all require visiting every element. If you understand traversal,
  you understand the skeleton of 80% of array algorithms.

HOW IT WORKS — STEP BY STEP:

  Given: LA = [10, 25, 30, 45, 50],  K = 5

  We use a pointer I that starts at 1 and moves forward to K.

  ┌───────┬──────────┬────────────────────────┐
  │ I     │ LA[I]    │ What happens           │
  ├───────┼──────────┼────────────────────────┤
  │ I = 1 │ LA[1]=10 │ Process 10 (print it)  │
  │ I = 2 │ LA[2]=25 │ Process 25             │
  │ I = 3 │ LA[3]=30 │ Process 30             │
  │ I = 4 │ LA[4]=45 │ Process 45             │
  │ I = 5 │ LA[5]=50 │ Process 50             │
  │ I = 6 │ 6 > K=5  │ STOP. Done.            │
  └───────┴──────────┴────────────────────────┘

  Output: 10  25  30  45  50

WHY DOES IT WORK?
  Because arrays are CONTIGUOUS (elements sit next to each other in memory).
  If you know the starting position and the count (K), you can access every
  element by simply incrementing the index. No element is skipped, no element
  is visited twice. The loop I = 1 to K guarantees exactly K visits.

THE PATTERN TO REMEMBER:
  "Start at 1. Go to K. Do something at each position."
  This pattern appears in searching, counting, summing, and more.


================================================================================
OPERATION 2: SEARCHING (Linear Search)
================================================================================

WHAT IT DOES:
  Find whether a specific ITEM exists in the array, and WHERE it is.

WHY YOU NEED IT:
  Before you can delete, update, or use an element, you first need to FIND it.
  Searching is the prerequisite for many other operations.

HOW IT WORKS — STEP BY STEP:

  Given: LA = [10, 25, 30, 45, 50],  K = 5,  ITEM = 30

  We traverse the array and COMPARE each element with ITEM.

  ┌───────┬──────────┬────────────────┬────────────────────────┐
  │ I     │ LA[I]    │ LA[I] = ITEM?  │ Action                 │
  ├───────┼──────────┼────────────────┼────────────────────────┤
  │ I = 1 │ LA[1]=10 │ 10 = 30? NO   │ Move to next           │
  │ I = 2 │ LA[2]=25 │ 25 = 30? NO   │ Move to next           │
  │ I = 3 │ LA[3]=30 │ 30 = 30? YES! │ FOUND at position 3!   │
  │       │          │               │ Set LOC := 3, STOP.    │
  └───────┴──────────┴────────────────┴────────────────────────┘

  What if ITEM = 99 (not in array)?

  ┌───────┬──────────┬────────────────┬────────────────────────┐
  │ I     │ LA[I]    │ LA[I] = ITEM?  │ Action                 │
  ├───────┼──────────┼────────────────┼────────────────────────┤
  │ I = 1 │ LA[1]=10 │ 10 = 99? NO   │ Move to next           │
  │ I = 2 │ LA[2]=25 │ 25 = 99? NO   │ Move to next           │
  │ I = 3 │ LA[3]=30 │ 30 = 99? NO   │ Move to next           │
  │ I = 4 │ LA[4]=45 │ 45 = 99? NO   │ Move to next           │
  │ I = 5 │ LA[5]=50 │ 50 = 99? NO   │ Move to next           │
  │ I = 6 │ 6 > K=5  │ —             │ Loop ended. NOT FOUND. │
  └───────┴──────────┴────────────────┴────────────────────────┘

WHY DOES IT WORK?
  By checking EVERY element, we guarantee we won't miss the item.
  If we reach the end without finding it, it's CERTAINLY not there.
  This is called LINEAR search because we go in a straight line
  from start to end — O(N) time.

WHY NOT START FROM THE END OR MIDDLE?
  You could! But it doesn't change the worst case. In the worst case
  (item not present), you ALWAYS have to check all K elements.
  Starting from position 1 is just the simplest convention.

THE KEY INSIGHT:
  Searching is just TRAVERSAL with a CONDITION. Instead of blindly
  processing every element, you CHECK each element against ITEM
  and STOP EARLY when you find a match.


================================================================================
OPERATION 3: INSERTION (at a given position)
================================================================================

WHAT IT DOES:
  Add a new ITEM at a specific position POS in the array.

WHY IT'S TRICKY:
  Unlike a linked list where you just rewire pointers, an array's elements
  are stored in CONTIGUOUS memory. You can't just "squeeze" a new element
  in — there's no gap. So you MUST CREATE a gap by SHIFTING elements.

HOW IT WORKS — STEP BY STEP:

  Given: LA = [10, 25, 30, 45, 50],  K = 5, N = 8
         Insert ITEM = 99 at POS = 3

  STEP A: Check if there's room (K < N?  5 < 8? YES, room exists)

  STEP B: We need to make space at position 3. That means positions
          3, 4, 5 all need to move ONE STEP TO THE RIGHT.

  BUT WHICH DIRECTION DO WE SHIFT?

  ╔═══════════════════════════════════════════════════════════════╗
  ║  CRITICAL: When shifting RIGHT, you MUST start from the END  ║
  ║  and work BACKWARDS. If you start from POS, you will DESTROY ║
  ║  data by overwriting it before moving it.                    ║
  ╚═══════════════════════════════════════════════════════════════╝

  WRONG WAY (starting from POS, going forward):
    Position: [1]  [2]  [3]  [4]  [5]  [6]
    Before:   10   25   30   45   50    _
    
    Copy LA[3] to LA[4]:  LA[4] = 30  → but 45 is DESTROYED!
    Position: [1]  [2]  [3]  [4]  [5]  [6]
              10   25   30   30   50    _     ← 45 is GONE forever!

  RIGHT WAY (starting from END, going backward):
    Position: [1]  [2]  [3]  [4]  [5]  [6]
    Before:   10   25   30   45   50    _

    ┌───────┬───────────────────┬──────────────────────────────────────┐
    │ I     │ Operation         │ Array state                          │
    ├───────┼───────────────────┼──────────────────────────────────────┤
    │ I = 5 │ LA[6] := LA[5]    │ [10, 25, 30, 45, 50, 50]            │
    │ I = 4 │ LA[5] := LA[4]    │ [10, 25, 30, 45, 45, 50]            │
    │ I = 3 │ LA[4] := LA[3]    │ [10, 25, 30, 30, 45, 50]            │
    │ I = 2 │ 2 < POS=3, STOP  │ [10, 25,  _, 30, 45, 50]  gap at 3! │
    └───────┴───────────────────┴──────────────────────────────────────┘

  STEP C: Place ITEM at POS: LA[3] := 99

    Result: [10, 25, 99, 30, 45, 50]

  STEP D: Update count: K := K + 1 = 6

WHY DOES IT WORK?
  By shifting from the END backward, each element is safely copied to
  the position AFTER it before it gets overwritten. No data is lost.
  The gap naturally appears at POS after all shifts are done.

VISUAL METAPHOR:
  Imagine 5 people sitting in a row of 8 chairs. You want person #99
  to sit in chair 3. You can't just put them there — someone is sitting!
  So you tell the person in chair 5 to move to chair 6 first. Then chair
  4 moves to 5. Then chair 3 moves to 4. NOW chair 3 is empty!

WHAT IF THE ARRAY IS FULL? (K = N)
  There's no chair to shift into! This is OVERFLOW. You MUST check
  for this BEFORE attempting any shifts.

TIME COMPLEXITY: O(N) — in the worst case (inserting at position 1),
  you have to shift ALL N elements.


================================================================================
OPERATION 4: DELETION (from a given position)
================================================================================

WHAT IT DOES:
  Remove the element at position POS and close the gap left behind.

WHY IT'S THE MIRROR OF INSERTION:
  Insertion creates a gap (shift right). Deletion fills a gap (shift left).
  If you understand one, you understand the other.

HOW IT WORKS — STEP BY STEP:

  Given: LA = [10, 25, 30, 45, 50],  K = 5
         Delete element at POS = 2

  STEP A: Check if array is empty (K = 0? NO)
          Check if POS is valid (1 <= POS <= K?  1 <= 2 <= 5? YES)

  STEP B: Save the element being deleted: ITEM := LA[2] = 25

  STEP C: Fill the gap by shifting everything AFTER POS to the LEFT.

  ╔═══════════════════════════════════════════════════════════════╗
  ║  CRITICAL: When shifting LEFT, start from POS and go FORWARD ║
  ║  This is the OPPOSITE of insertion!                          ║
  ╚═══════════════════════════════════════════════════════════════╝

    ┌───────┬───────────────────┬──────────────────────────────────┐
    │ I     │ Operation         │ Array state                      │
    ├───────┼───────────────────┼──────────────────────────────────┤
    │ I = 2 │ LA[2] := LA[3]    │ [10, 30, 30, 45, 50]             │
    │ I = 3 │ LA[3] := LA[4]    │ [10, 30, 45, 45, 50]             │
    │ I = 4 │ LA[4] := LA[5]    │ [10, 30, 45, 50, 50]             │
    │ I = 5 │ 5 >= K=5, STOP   │ [10, 30, 45, 50, _]  last is junk│
    └───────┴───────────────────┴──────────────────────────────────┘

  STEP D: Update count: K := K - 1 = 4

    Result: [10, 30, 45, 50]  (the "50" at position 5 is now irrelevant
            because K = 4, so we only consider positions 1 to 4)

WHY SHIFT FORWARD FOR DELETION?
  When you remove an element, positions AFTER it need to fill the gap.
  Starting from POS and going forward means each element copies the
  value from the position AFTER it (pulling everything left). The last
  position becomes "junk" but we ignore it because K decreases by 1.

VISUAL METAPHOR:
  5 people sitting in chairs. Person in chair 2 leaves. Now there's a gap.
  Person in chair 3 slides to chair 2. Person in chair 4 slides to 3.
  Person in chair 5 slides to 4. Chair 5 is now empty, but we don't care
  because our headcount dropped from 5 to 4.

INSERTION vs DELETION — THE SHIFT RULE:
  ┌──────────────────────────────────────────────────────────────┐
  │  INSERTION (shift RIGHT):  Start from END,  go BACKWARD     │
  │  DELETION  (shift LEFT):   Start from POS,  go FORWARD      │
  │                                                              │
  │  Why? Because you must SAVE data before overwriting it.      │
  │  Always move elements AWAY from the direction of shift first.│
  └──────────────────────────────────────────────────────────────┘

OPERATION 5: BUBBLE SORT
================================================================================

WHAT IT DOES:
  Arrange all elements in ascending (or descending) order.

WHY IT'S CALLED "BUBBLE" SORT:
  In each pass, the LARGEST unsorted element "bubbles up" to its correct
  position at the end — like a bubble rising to the surface of water.

HOW IT WORKS — STEP BY STEP:

  Given: LA = [50, 30, 10, 40, 20],  K = 5

  THE IDEA: Compare adjacent pairs. If they're in the wrong order, SWAP.
  Repeat this for the entire array. After each pass, the largest unsorted
  element will have "bubbled" to the end.

  PASS 1 (K=1, compare positions 1 to N-1=4):
  ┌───────────┬──────────────┬─────────┬────────────────────────┐
  │ Compare   │ Values       │ Swap?   │ Array after            │
  ├───────────┼──────────────┼─────────┼────────────────────────┤
  │ LA[1]:LA[2]│ 50 > 30?  YES│ SWAP    │ [30, 50, 10, 40, 20]   │
  │ LA[2]:LA[3]│ 50 > 10?  YES│ SWAP    │ [30, 10, 50, 40, 20]   │
  │ LA[3]:LA[4]│ 50 > 40?  YES│ SWAP    │ [30, 10, 40, 50, 20]   │
  │ LA[4]:LA[5]│ 50 > 20?  YES│ SWAP    │ [30, 10, 40, 20, 50]   │
  └───────────┴──────────────┴─────────┴────────────────────────┘
  After Pass 1: 50 is in its correct position (position 5) ✓

  PASS 2 (K=2, compare positions 1 to N-2=3):
  ┌───────────┬──────────────┬─────────┬────────────────────────┐
  │ Compare   │ Values       │ Swap?   │ Array after            │
  ├───────────┼──────────────┼─────────┼────────────────────────┤
  │ LA[1]:LA[2]│ 30 > 10?  YES│ SWAP    │ [10, 30, 40, 20, 50]   │
  │ LA[2]:LA[3]│ 30 > 40?  NO │ —       │ [10, 30, 40, 20, 50]   │
  │ LA[3]:LA[4]│ 40 > 20?  YES│ SWAP    │ [10, 30, 20, 40, 50]   │
  └───────────┴──────────────┴─────────┴────────────────────────┘
  After Pass 2: 40 is in its correct position (position 4) ✓

  PASS 3 (K=3, compare positions 1 to N-3=2):
  ┌───────────┬──────────────┬─────────┬────────────────────────┐
  │ Compare   │ Values       │ Swap?   │ Array after            │
  ├───────────┼──────────────┼─────────┼────────────────────────┤
  │ LA[1]:LA[2]│ 10 > 30?  NO │ —       │ [10, 30, 20, 40, 50]   │
  │ LA[2]:LA[3]│ 30 > 20?  YES│ SWAP    │ [10, 20, 30, 40, 50]   │
  └───────────┴──────────────┴─────────┴────────────────────────┘
  After Pass 3: 30 is in its correct position (position 3) ✓

  PASS 4 (K=4, compare positions 1 to N-4=1):
  ┌───────────┬──────────────┬─────────┬────────────────────────┐
  │ Compare   │ Values       │ Swap?   │ Array after            │
  ├───────────┼──────────────┼─────────┼────────────────────────┤
  │ LA[1]:LA[2]│ 10 > 20?  NO │ —       │ [10, 20, 30, 40, 50]   │
  └───────────┴──────────────┴─────────┴────────────────────────┘
  After Pass 4: All elements sorted ✓

  FINAL: [10, 20, 30, 40, 50]

WHY DOES IT WORK?
  In each pass, the largest unsorted element is guaranteed to reach
  its correct position (at the right end of the unsorted portion).
  After K-1 passes, all elements are in their correct positions.

  Why K-1 passes? Because after K-1 elements are placed correctly,
  the remaining 1 element must also be in the right place (there's
  nowhere else for it to go!)

WHY DOES EACH PASS GET SHORTER?
  After Pass 1, position K is sorted → don't compare it again.
  After Pass 2, positions K and K-1 are sorted → skip those.
  The inner loop runs N-K comparisons in pass K.

THE SWAP MECHANISM (3 steps):
  To swap LA[I] and LA[I+1]:
    TEMP := LA[I]          ← save first value
    LA[I] := LA[I+1]       ← overwrite with second
    LA[I+1] := TEMP        ← put saved value in second's place

  Why do we need TEMP? Because if we directly do LA[I] := LA[I+1],
  the original value of LA[I] is LOST forever. TEMP saves it.

TIME COMPLEXITY:
  Worst case:  O(N²) — every pair is out of order
  Best case:   O(N²) — even if sorted, we still do all comparisons
               (unless we add an optimization flag)

OPERATION 6: SUM OF ARRAY ELEMENTS
================================================================================

WHAT IT DOES:
  Add up all elements and return the total.

HOW IT WORKS — STEP BY STEP:

  Given: LA = [10, 25, 30, 45, 50],  K = 5

  We use a variable SUM starting at 0. At each position, we ADD
  the current element to SUM.

  ┌───────┬──────────┬────────────────────────┬─────────┐
  │ I     │ LA[I]    │ Operation              │ SUM     │
  ├───────┼──────────┼────────────────────────┼─────────┤
  │       │          │ Initialize             │ 0       │
  │ I = 1 │ LA[1]=10 │ SUM := 0 + 10          │ 10      │
  │ I = 2 │ LA[2]=25 │ SUM := 10 + 25         │ 35      │
  │ I = 3 │ LA[3]=30 │ SUM := 35 + 30         │ 65      │
  │ I = 4 │ LA[4]=45 │ SUM := 65 + 45         │ 110     │
  │ I = 5 │ LA[5]=50 │ SUM := 110 + 50        │ 160     │
  └───────┴──────────┴────────────────────────┴─────────┘

  Result: SUM = 160

WHY START SUM AT 0?
  Because 0 is the IDENTITY element for addition (adding 0 to anything
  doesn't change it). If we started SUM at some random value, our total
  would be wrong. This is a common pattern:
    Sum   → start at 0
    Count → start at 0
    Max   → start at first element
    Min   → start at first element


================================================================================
THE BIG PICTURE: HOW ALL ARRAY OPERATIONS CONNECT
================================================================================

  ┌─────────────┐
  │  TRAVERSAL  │──→ The FOUNDATION. Every other operation is built on it.
  └──────┬──────┘
         │
    ┌────┴────┐
    │         │
    ▼         ▼
┌────────┐ ┌────────────┐
│ SEARCH │ │ SUM/COUNT  │
│        │ │ MAX/MIN    │  ← These are all "traversal with a purpose"
└───┬────┘ └────────────┘
    │
    ▼
┌─────────────────────────┐
│ INSERT / DELETE          │  ← These require SHIFTING elements
│ (the core CRUD ops)     │
└───────────┬─────────────┘
            │
            ▼
     ┌──────────────┐
     │ BUBBLE SORT  │  ← Uses COMPARISON + SWAP in nested loops
     └──────────────┘

  KEY TAKEAWAY:
  If you deeply understand TRAVERSAL and SHIFTING, you can build
  every single array algorithm from scratch. They are the two
  fundamental building blocks.
